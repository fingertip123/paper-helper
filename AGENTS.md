# AGENTS.md · 研栈 Agent 工作规范

本仓库是一个**基于 Karpathy「LLM Wiki」范式**的个人知识库。
分工原则：**人类负责策展（curate），Agent 负责簿记（maintain）**。

> 开始任何操作前，先读 `purpose.md`（方向）和 `schema.md`（结构规则）。

---

## 三个核心操作

### 1. Ingest（摄入）—— 把原始资料编译成 Wiki

当用户把文献/笔记放入 `raw/sources/`，或说“摄入这篇/这份资料”时：

**第一步：分析**
- 读取原始资料，提取：关键实体、概念、论点、方法、数据、结论。
- 对照已有 `wiki/` 内容，找出关联点与**矛盾/张力**。
- 给文献分配引用 key（见 `schema.md` §5）。

**第二步：生成 / 更新**
1. 在 `wiki/sources/<key>.md` 写结构化摘要（带 frontmatter，含 `sources: [<key>]`）。
2. 新建或更新 `wiki/concepts/`、`wiki/entities/` 等相关页面。
3. 一次性补全所有相关页面的 `[[wikilink]]` 交叉引用。
4. 在 `wiki/log.md` 追加一条审计记录。
5. 重写 `wiki/overview.md` 反映最新全局状态。
6. 把需要人工判断的项（矛盾、存疑、待核实）列在回复末尾的「审核区」。

> 注：`wiki/index.md` 现由工具（`tools/wiki_core.py` 的 `GenerateIndex`）自动生成，
> 每次通过本地服务 添加/分析/刷新 时刷新，无需手动维护。

### 2. Query（查询）—— 基于已编译知识回答

当用户提问时：
1. 读 `purpose.md` + `wiki/index.md` 定位相关页面。
2. 沿 `[[wikilink]]` 扩展 1–2 跳，收集相关页面的**完整内容**。
3. 作答时按页面编号引用，如 `[1] [[transformer]]`。
4. 若回答产生了有价值的新洞察，询问/直接沉淀到 `wiki/queries/`，并按需触发一次轻量 Ingest 把新概念并入网络。

### 3. Lint（巡检）—— 保持知识库健康

当用户说“巡检 / lint / 体检”时，检查并报告：
- **孤立页面**：没有任何入链或出链的页面。
- **死链**：指向不存在页面的 `[[wikilink]]`。
- **缺失 frontmatter / 字段**：不符合 `schema.md` 的页面。
- **知识空白**：`purpose.md` 的研究问题中尚无对应页面覆盖的部分。
- **index/overview 失真**：与实际文件不一致之处。

报告后，在用户确认下修复（补链、归档、删除、新建占位页）。

---

## 通用约束

- 严禁修改 `raw/` 下的文件（只读来源层）。
- 每次写操作都要**同步更新** `index.md` 与 `log.md`。
- 代码/文件命名遵循 `schema.md`；不臆造文献内容。
- 回复用简体中文，简洁精确。
