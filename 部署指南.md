# 研栈 · 云端多用户部署指南

代码已支持云端多用户：注册/登录、每用户数据完全隔离（`data/u-<id>/`）、
共享内置 GLM Key 每人每日限额（默认 60 次，可配自己的 Key 解除）。

> 注册账号 / 域名需要你本人的邮箱验证，无法由 AI 代办；按下面步骤约 15 分钟完成。

---

## 关于「永久免费域名」

真正免费的顶级域名（如以前的 Freenom .tk）已基本消亡。现实可行的永久免费方案：

| 方案 | 你得到的网址 | 说明 |
|------|-------------|------|
| PythonAnywhere（推荐） | `你的用户名.pythonanywhere.com` | 托管+域名一体，免费版永久 |
| DuckDNS | `你起的名.duckdns.org` | 免费子域名，配合自己的服务器用 |
| eu.org | `你起的名.eu.org` | 真正的免费域名，但人工审核需数周 |

## 方案 A：PythonAnywhere（推荐：免费、磁盘持久、不睡眠）

1. 到 <https://www.pythonanywhere.com> 注册免费账号（Beginner，永久免费，512MB 磁盘）。
2. 打开 **Consoles → Bash**，拉取代码并装依赖：

   ```bash
   git clone <你的仓库地址> paper-helper
   cd paper-helper
   pip3 cache purge
   pip3 install --user --no-cache-dir -r requirements-server.txt
   ```

   **磁盘配额（Errno 122）**：免费账号 home 目录总共约 **512MB**（含代码、pip 缓存、`data/`、用户文献）。
   若 `pip install` 报 `Disk quota exceeded`：

   - 先执行 `pip3 cache purge`（pip 缓存常占 100MB+）
   - 在 **Files** 页删除不需要的大目录：`release/`、`dist/`、`build/`、`.venv/`、`Yanzhan.app/`（云端用不到）
   - 文献 PDF 放 `data/u-*/topics/.../raw/sources/`，体量大会很快顶满；免费版只适合轻量试用
   - 查看占用：`du -sh ~/* ~/paper-helper/* | sort -hr | head -20`
   - 安装时**必须**加 `--no-cache-dir`，不要装 `requirements.txt`（含 PySide6，体积巨大）

   （仓库私有的话，用 Files 页上传 zip 再解压也可以；zip 里不要打包 `release/`、`dist/`、`topics/` 里的 PDF。）

3. **重要**：把内置 GLM Key 放上去（该文件已 gitignore，不在仓库里）：
   在 Files 页新建 `/home/<用户名>/paper-helper/tools/builtin_api_key.txt`，
   内容粘贴你的智谱 API Key 一行。

4. 打开 **Web → Add a new web app → Manual configuration → Python 3.10+**。
5. 点开 **WSGI configuration file**，全部替换为：

   ```python
   import sys
   sys.path.insert(0, "/home/<你的用户名>/paper-helper")
   from wsgi import application
   ```

6. 点 **Reload**。访问 `https://<你的用户名>.pythonanywhere.com` 即可注册使用。

数据存在 `paper-helper/data/`（磁盘持久，重启不丢）。
限制：免费版出站网络走白名单，`open.bigmodel.cn` 如不在白名单内可在论坛申请加白，
或让用户改用已加白的 OpenAI 兼容端点。

## 方案 B：自己的服务器（Oracle Cloud 永久免费 VM 等）+ DuckDNS

1. Oracle Cloud「Always Free」可申请永久免费 ARM VM（需信用卡验证身份，不扣费）。
2. VM 上跑：

   ```bash
   git clone <仓库> paper-helper && cd paper-helper
   pip3 install -r requirements-server.txt
   # 写入 tools/builtin_api_key.txt（内容为你的智谱 Key）
   YANZHAN_DATA_DIR=/srv/yanzhan-data nohup python3 tools/server.py &
   ```

   或用 Docker：`docker build -t yanzhan . && docker run -d -p 80:8765 -v /srv/yanzhan-data:/srv/yanzhan/data yanzhan`

3. 到 <https://www.duckdns.org>（GitHub 账号即可登录）建一个子域名，把 IP 指到 VM 公网 IP。
4. 可选：用 Caddy 一行配置自动 HTTPS：`caddy reverse-proxy --from 你的名.duckdns.org --to :8765`

## 方案 C：Render / Koyeb（部署最简单，但免费版磁盘不持久 ⚠️）

仓库已含 `Dockerfile`，在 Render 新建 Web Service 指向仓库即可。
**注意**：免费实例重启后用户数据会丢失，只适合演示，不适合正式使用。

## 环境变量

| 变量 | 默认 | 说明 |
|------|------|------|
| `PORT` / `HOST` | 8765 / 0.0.0.0 | 监听地址 |
| `YANZHAN_DATA_DIR` | `<项目>/data` | 用户数据根（生产务必指向持久盘） |
| `YANZHAN_MULTIUSER` | 1 | 设 0 退化为单用户实例 |
| `YANZHAN_LLM_DAILY` | 60 | 共享内置 Key 时每人每日调用上限，0 不限 |

## 本地验证多用户模式

```bash
YANZHAN_DATA_DIR=/tmp/yz-data python3 tools/server.py
# 浏览器打开 http://127.0.0.1:8765 → 注册两个账号验证数据互不可见
```
